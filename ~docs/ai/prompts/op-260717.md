# RAG Application Architecture Specification

建立與加上 前後端 的 sso 呼叫，分離到 lib 中，前端儲存目錄 lib/react/sso 套件名為 eiva_fe_sso；後端儲存目錄 lib/rust/sso, 套件為 eiva_be_sso

##

建立與加上 前後端 的 使用者，角色與權限管理 功能，
    - 分離到 lib 中，
    - 前端儲存目錄 lib/react/common 套件名為 eiva_fe_security；
    - 後端儲存目錄 lib/rust/common, 套件為 eiva_be_security
前後端的 env 中，加入以下機制
    - login_type : 分為 mock, standalone, sso_keyclock
        - mock: 給開發人員測試用，使用 root 帳號, 密碼: mithra35
        - standalone: 使用 後端 資料庫，建立 使用者，角色，權限 進行身分驗證
        - sso_keyclock: 使用 Keycloak 作為 SSO 伺服器，進行身分驗證
    - 依 login_type 進行前後端登入與認證機制切換
    - 前端建立 以下畫面，相關後端認證流程規格如下
        - 前端源碼 儲存於 eiva_fe_security 中
        - 後端源碼 儲存於 eiva_be_security 中
        - 相關前端畫面建立如下:
            - 登入畫面，依 login_type 進行處理
            - 使用者資料管理畫面，顯示與編輯使用者資訊、角色與權限設定
                - 使用者列表與搜尋功能，支持批次操作與過濾查詢
            - 角色與權限設定畫面，管理角色定義與權限映射
                - 角色列表與搜尋功能，支持批次操作與過濾查詢
                - 角色詳細資訊與編輯功能
                - 權限映射，選單與分配功能
            - 使用者詳細資訊與編輯畫面
            - 密碼重置與修改畫面
            - 個人設定與安全設定畫面
    - 整合 系統與 SSO 認證機制
        - 建立 Keycloak 配置文件和客戶端設定
        - 實作 OAuth 2.0 / OpenID Connect 流程
        - 處理 SSO 回傳的 JWT Token 和使用者資訊
        - 建立 Session 管理與 Token 刷新機制
        - 整合前後端的認證中間件與守護層
